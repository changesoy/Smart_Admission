# 智慧入学·学区治理一站式可视化门户 — 项目交接文档

> 本文档汇总了项目背景、目标、技术决策、当前实现状态、源数据参考、最新变更与待办事项,可在新对话中作为参考材料直接使用。

---

## 一、项目概述与目标(重点)

### 1.1 项目背景

随着我国数字政府建设全面推进,政务服务网已形成"政务公开、政务服务、政民互动、数据服务、专项服务、效能监督"六大体系。但在教育民生领域,义务教育入学、学区划分、学位资源分配是社会关注的核心痛点,存在以下问题:

- **渠道分散**:学区划分、招生政策、学位数量散落在教育局官网、学校公众号、社区通知等多个平台
- **口径不一、更新滞后**:历史政策无统一归档,新老政策对比困难
- **规则不透明**:入学积分规则、录取顺位不公开,易引发谣言与焦虑
- **咨询成本高**:家长只能通过电话、线下窗口咨询

目前地方政务服务网尚未形成统一、专属、全周期的入学招生信息服务平台,现有教育板块仅简单嵌入政策文件,未实现学区地图、学位预警等精细化服务。

### 1.2 项目目标

设计一款**针对学区与入学招生信息查询的专用政务网站**,契合数字政府便民服务的发展方向:

1. **解决民生痛点**:整合碎片化教育信息,实现一站式查询,缓解升学焦虑
2. **提升政务效率**:将线下及电话咨询转为线上自助服务,降低教育部门窗口压力
3. **验证技术路线**:通过"前端 GIS + 本地 JSON/GeoJSON"的轻量化架构,实现高可迁移、低成本的政务网站方案(更换底层数据文件即可适配新城市,前端零代码修改)

### 1.3 当前阶段定位

**当前为阶段性本地前端原型**(非生产系统):

- 仅含前端代码与示例数据
- 无后端、无数据库、无登录注册、无真实在线咨询功能
- 学区边界为示例或简化示意数据,不代表官方学区划分
- 示例数据仅用于展示技术路线,实际入学政策以教育主管部门官方发布为准
- 选用山东省泰安市泰山区作为示范区域(因有公开的学区四至范围参考资料)

## 二、功能设计(来自选题报告)

### 2.1 核心功能

1. **学区一键查询**
   - 支持小区名称、街道、路名关键词检索,自动匹配对口小学、初中
   - 提供文字学区范围 + 地图边界标注
   - 标注学区历年调整记录、调整原因,实现信息可追溯
2. **招生政策总览**
   - 公示当年义务教育招生日程表、招生对象、年龄要求、户籍规则
   - 置顶当年政策变化点,避免家长误解
3. **录取建议模拟**
   - 对照往年积分分数线,给出择校参考建议

### 2.2 基础功能

1. **入学材料清单**:按本地户籍、随迁子女、集体户、人才子女分类,公示所需材料
2. **历史政策归档**:归档近 3-5 年学区划分图、招生政策、录取分数线
3. **常见问题(FAQ)**:整理家长高频咨询问题,提供自动问答

### 2.3 辅助功能

1. **政策解读与辟谣**:发布通俗化政策解读,澄清网络不实信息
2. **政民互动**:提供在线留言、咨询入口

## 三、技术架构(严格锁定,不可变更)

### 3.1 必须使用的技术栈

| 类别     | 技术                                                        |
| -------- | ----------------------------------------------------------- |
| 基础     | HTML5 / CSS3 / Vanilla JavaScript(无任何框架)               |
| UI 框架  | Bootstrap 5.3.2 + Bootstrap Icons 1.11.3                    |
| 地图     | Leaflet 1.9.4                                               |
| 空间计算 | Turf 6.5.0(`turf.booleanPointInPolygon`)                    |
| 数据     | 本地 JSON / GeoJSON 文件(通过 fetch 加载)                   |
| 底图     | **天地图(已切换,原 OSM)** — 矢量底图 vec_w + 矢量注记 cva_w |
| 运行     | 本地 HTTP 服务(如 `python -m http.server 5500`)             |

| 图层     | layer 参数 | 说明             |
| -------- | ---------- | ---------------- |
| 矢量底图 | `vec_w`    | 街道矢量(主底图) |

**核心 URL 格式**:

```
https://t{s}.tianditu.gov.cn/vec_w/wmts?SERVICE=WMTS&REQUEST=GetTile&VERSION=1.0.0&LAYER=vec&STYLE=default&TILEMATRIXSET=w&FORMAT=tiles&TILEMATRIX={z}&TILEROW={y}&TILECOL={x}&tk={token}
```

### 6.2 布局

- 桌面端(≥ 992px):地图与结果面板左右排列,地图 65%,面板 35%
- 卡片式布局,阴影 `box-shadow: 0 2px 8px rgba(0,0,0,0.06)`
- 地图容器最小高度桌面端 560px
- 圆角统一 8px(卡片),按钮圆角 6px
- 风格:政务后台管理系统(参考 Element Plus、Ant Design Pro)

### 6.3 关键交互细节

1. **坐标顺序**(极易混淆):
   - GeoJSON `coordinates`:`[经度, 纬度]` (lng, lat)
   - Leaflet `setView`/`latlng`:`[纬度, 经度]` (lat, lng)
   - `turf.point()`:`[经度, 纬度]` (lng, lat)
   - Leaflet `e.latlng` → 传给 Turf 时:`[e.latlng.lng, e.latlng.lat]`
2. **学区点击必须** **`L.DomEvent.stopPropagation(e)`** 阻止冒泡到地图空白点击逻辑
3. **同一时刻只有一个学区选中态**,点击新学区时旧的恢复默认
4. **材料勾选不持久化**,刷新即重置

## 八、常见报错与排查(已整理)

| 报错现象                                      | 原因                                    | 解决办法                                                       |
| --------------------------------------------- | --------------------------------------- | -------------------------------------------------------------- |
| 停在"数据加载中…"/`Failed to fetch`/CORS 报错 | 双击了 `index.html`,使用 `file://` 协议 | 执行 `python -m http.server 5500`,改用 `http://localhost:5500` |
| `L is not defined`/`turf is not defined`      | CDN 无法访问                            | 切换网络;或下载到本地 `vendor/` 目录                           |
| 地图区域灰色、看不到瓦片                      | 无法访问天地图瓦片                      | 检查 token 是否绑定域名;localhost 通常默认放行                 |
| 401/403 错误                                  | token 错误/过期/未绑定域名              | 在天地图开发者控制台检查                                       |
| 点击没反应/Polygon 不渲染                     | GeoJSON 加载失败或格式错误              | 检查 Network 面板;用 <https://geojson.io> 校验                 |
| 点击空白却命中学区                            | 坐标顺序写反                            | GeoJSON/Turf 用 `[lng, lat]`;Leaflet 用 `[lat, lng]`           |
| 学区字段显示 "—"                              | schoolId/policyId 在 JSON 中不存在      | 对照 ID 关联校验表逐一检查                                     |
| 混合内容警告                                  | HTTP 页面加载 HTTPS 资源                | 确保协议一致,生产环境必须 HTTPS                                |

## 十、关键约束总结(防止丢失)

1. ❌ **绝不使用** Vue/React/npm/构建工具/ES Module/TypeScript
2. ❌ **绝不让用户双击 HTML 运行**,必须用 HTTP 服务
3. ✅ **必须** 使用本地 JSON/GeoJSON 驱动,前端零代码迁移
4. ✅ **必须** 严格遵守锁定的 CDN 版本号
5. ✅ **必须** 标注"示例数据·非官方"声明
6. ✅ **必须** 在结果面板做 `undefined` 防御性编程
7. ✅ **必须** 在学区点击时 `L.DomEvent.stopPropagation(e)`
8. ✅ **必须** 注意 GeoJSON `[lng, lat]` vs Leaflet `[lat, lng]` 的坐标顺序差异
